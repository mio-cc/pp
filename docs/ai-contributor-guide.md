# AI 贡献者指南 — AI视觉设计与提示词工程百科

本文档供 AI 模型（Claude、GPT、Gemini 等）阅读，以便理解项目结构并批量填充术语数据。

## 1. 项目概述

本项目是一个面向约 **10,000 条**专业视觉设计术语的结构化知识库，横跨 15 个卷册，服务两个核心目标：

1. **百科解释** — 每条术语提供中英文名、定义、视觉表现描述
2. **AI 生图提示词构建** — 每条术语提供可直接用于 Stable Diffusion / Midjourney / DALL-E 等模型的英文提示词片段

## 0. 核心规范更新（V3 · 必读）

向本项目补充术语/卷之前，**必须**遵守以下规则，否则数据会被校验拒绝或破坏一致性：

1. **分类是「路径」，深度不限**：`category` 字段写成从属路径，用 ` / `（空格-斜杠-空格）分隔，例如
   `镜头与景深 / 光学缺陷 / 镜头畸变`。系统按斜杠递归建树，层数完全由领域需要决定（2 层、3 层、4 层均可）。
2. **术语必须原子化（靠分支拆分）**：一个术语 = 一个明确、可直接复制、不产生歧义的概念。
   笼统词要沿路径继续分支成具体值。例如不要只写「镜头畸变」，而要拆成
   `… / 镜头畸变` 下的 `桶形畸变`、`枕形畸变`；焦距拆成 `… / 定焦镜头` 下的 `35mm广角`、`50mm标准`、`85mm人像`。
3. **同一最深分支 = 互斥可选项**：放在同一叶子路径下的术语天然互斥（前端篮中若同时出现会标红提示）。
   因此「冲突规避」不靠字段，而靠把不同属性/取值分到不同分支。
4. **提示词中英双语并存、与术语对应（不用负向）**：
   - 每条术语只配**一个**"提示词"，填 `positive_prompt`(英文) + `positive_prompt_cn`(中文)，二者一一对应到该术语本身。
     例：冷蓝色调 → `cool blue tone` / `冷蓝色调`；85mm人像 → `85mm lens, portrait` / `85mm镜头, 人像`。
   - **不使用负向提示词**：`negative_prompt` / `negative_prompt_cn` 两列保留但**留空**，前端不展示。
5. **不要新增互斥组等字段**；不要在 API 端做冲突检测（冲突仅为前端视觉提示）。
6. **链路必须干净**：中间层是「分类/分支」，末端是「原子术语/叶子」。同一层级不要把概述词与子分类并列。
   反例：在「焦距与视角」下同时放 `焦距`(术语) 和 `定焦镜头`(子类)。
   正解：`焦距 / 定焦镜头 / 50mm标准`——点「焦距」才看到定焦/变焦，再点进去才是 35/50/85mm。
7. **提示词取最低层原子值**：生图用的术语必须是最具体的叶子，而非笼统概念。
   反例 `焦距`、`颜色`、`色温`；正例 `85mm人像`、`深红色`、`暖色温3200K`。
   颜色还要细到深浅浓淡，如 `浅天蓝`、`浓郁勃艮第红`。
8. **提示词内容 = 术语本身**：`positive_prompt_cn` 即该术语中文（如「冷蓝色调」），`positive_prompt` 是其英文（如 `cool blue tone`）。原子值术语通常无负向，留空即可。

## 5. Term UID 编号规则

格式：`V{卷册编号}_T{四位序号}`

- 卷册编号：两位数字，如 V01、V15
- 序号：四位数字，从 0001 开始递增
- 每卷独立编号，例如 V01_T0001 ~ V01_T0500

**重要**：追加新术语时，先查看当前 CSV 中该卷的最大序号，从下一个开始。例如当前 V01 最大是 V01_T0006，新增术语应从 V01_T0007 开始。

## 6. 质量标准

### 6.1 双重服务原则

每条术语必须同时服务于：
- **百科功能**：definition_short / definition_long / visual_effect 要准确、专业、有教育价值
- **提示词功能**：positive_prompt 必须是真实可用的英文提示词片段，能直接粘贴到 SD/MJ 中产生对应视觉效果

### 6.2 正向提示词要求

- 使用英文
- 是可直接使用的提示词片段，不是解释性文字
- 可包含多个逗号分隔的关键词组合
- 示例：`portrait, shallow depth of field, soft background, 85mm lens`

### 6.3 负向提示词要求

- 使用英文
- 列出使用该术语时应避免的视觉元素
- 用分号分隔多个条目
- 示例：`blurry subject;unwanted bokeh`

### 6.4 分类一致性

category 第一段必须是 config/volumes.json 中该卷定义的顶层分类；其后可用 ` / ` 继续分支到更细的子类/原子层（深度不限）。把不同取值/属性分到不同分支，确保每个叶子术语是可直接复制、互不冲突的原子项。

## 10. 常见错误

| 错误 | 说明 |
|------|------|
| 多值分隔符用逗号 | 必须用英文分号 `;`，逗号在 CSV 中是字段分隔符 |
| term_uid 重复 | 每条术语的 UID 必须唯一，新增前检查最大编号 |
| category 不匹配 | 必须使用 config/volumes.json 中定义的分类名 |
| 只填英文提示词 | 必须中英双语并存：positive_prompt(英) + positive_prompt_cn(中) 都要填，且语义对应 |
| 术语太笼统 | 必须原子化：沿 category 路径分支到具体值(如 桶形畸变/枕形畸变)，而非只写「镜头畸变」 |
| category 用逗号或单层 | category 是 ` / ` 分隔的路径，深度不限 |
| 正向提示词写成解释 | 应该是可直接使用的关键词组合，不是句子 |
| 缺少必填字段 | 所有标记为"必填"的字段都不能为空 |
| 状态值拼错 | 只接受 draft/review/published/deprecated |
| CSV 行内换行 | 每条术语必须在单行内完成，不能换行 |

## 11. 建议的贡献策略

1. **按卷册批量填充**：一次专注填充一个卷册的所有分类
2. **先覆盖再深入**：先确保每个分类都有基本术语，再逐步增加深度
3. **参考专业资料**：术语定义应准确专业，不要编造不存在的术语
4. **测试提示词**：理想情况下，positive_prompt 应该是经过验证能产生对应视觉效果的提示词
5. **保持风格一致**：与现有 90 条种子术语保持同等的描述深度和格式风格
