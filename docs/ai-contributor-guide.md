# AI 供稿指南 v2 — AI视觉设计与提示词工程百科

> 本文件供 AI 模型（Claude / GPT / Gemini 等）阅读，用于**按规范批量补全术语**。
> 读完即可上手。**不要手改 CSV**；通过 `scripts/ingest.py` 接口提交 JSON，工具会校验后入库。

---

## 一、项目与核心理念

面向 15 卷、目标约 1 万条专业「视觉/图像」术语的结构化知识库。每条术语同时服务两件事：
1. **百科**：中英文名 + 详细解释 + 视觉表现。
2. **AI 生图提示词**：可直接粘进 Stable Diffusion / Midjourney / DALL·E 的提示词。

**核心理念：原子化术语的名字本身就是提示词。** `zh_term` 就是中文提示词，`en_term` 就是英文提示词。
所以 **没有** `positive_prompt` 这种列（v2 已删）。你要保证术语名足够原子、具体，本身就能直接当提示词用。

数据流：`你的 JSON` →(`scripts/ingest.py`)→ `data/raw/terms_seed.csv` →(`rebuild`)→ SQLite 主库 → API / 前端。

## 二、唯一写入方式：ingest 接口（务必照此做）

不要直接编辑 `terms_seed.csv`（手改极易列错位、漏转义、漏字段）。改为产出 JSON，交给接口：

```bash
python scripts/ingest.py check      你的术语.json     # 第一步：只校验不写库，看有没有错
python scripts/ingest.py add-terms  你的术语.json     # 校验全过 → 写入 → 重建 → 再校验
python scripts/ingest.py add-volume 新卷.json         # 新增一个卷（体系）
```

- `你的术语.json`：**对象数组**，每个对象是一条术语，字段见第三节，模版见 `docs/templates/term.template.json`，机器规范见 `schema/term.schema.json`。
- 多值字段（aliases / use_cases / related_terms / confused_with / tags）一律写**数组** `["a","b"]`，**不要写分号**（接口会自动转成 CSV 的 `;`）。
- `term_uid` 可以**留空**（写 `""` 或不写该键），接口会按该卷现有最大号 +1 自动分配。
- 先跑 `check`，确认输出「✓ 校验通过」再跑 `add-terms`。任何一条不合规，整批都不会写入，并逐条告诉你哪里错。

### 三种填充场景，全部用 `add-terms`，区别只在 `category` 怎么写

| 场景 | 怎么做 |
|------|--------|
| **给已有分支加原子术语** | `category` 写到已存在的叶子路径，例 `"曝光控制 / 快门速度"`。 |
| **给已有卷加新分支** | `category` 写一条更深的新路径即可（分支会自动创建），但**首段仍须是该卷在 config 里的顶层分类**。例 `"曝光控制 / 快门速度 / 高速快门 / 1-4000秒"`。 |
| **新增整卷（新体系）** | 先 `add-volume 新卷.json` 注册卷与顶层分类，再 `add-terms` 给它填术语。 |

## 三、字段逐条精确说明（共 16 个；标「必填」的不能空）

> 对每个字段，下面给出「**是什么 / 怎么写 / 正例 / 反例**」。务必照此填，不要自由发挥。

1. **term_uid**（选填，留空自动分配）
   - 是什么：术语唯一编号，格式 `V` + 两位卷号 + `_T` + 四位序号，如 `V01_T0193`。
   - 怎么写：通常**留空**让接口分配；若手填，必须卷内唯一、前缀与 `volume_code` 一致。
   - 正例：`""`（推荐）或 `"V06_T0241"`。 反例：`"T1"`、`"V6_1"`、与已有重复。

2. **zh_term**（必填 = 中文提示词）
   - 是什么：术语的中文名，也是中文提示词本身。必须是**原子最小单位**。
   - 怎么写：具体到可直接出图的取值；笼统概念要拆细。
   - 正例：`"85mm人像"`、`"深红色"`、`"暖色温3200K"`、`"桶形畸变"`。
   - 反例：`"焦距"`、`"颜色"`、`"色温"`、`"镜头畸变"`（这些是分类/分支，不是术语）。

3. **en_term**（必填 = 英文提示词）
   - 是什么：术语英文名，也是英文提示词，能直接粘进绘图工具。
   - 怎么写：用通行英文表述；与 zh_term 对应。
   - 正例：`"85mm Portrait Lens"`、`"Deep Red"`、`"Barrel Distortion"`。 反例：留空、机翻乱码。

4. **aliases**（选填，数组）
   - 是什么：别名/同义说法，仅用于搜索命中，不在卡片展示。
   - 怎么写：数组，可空。 正例：`["中长焦人像"]`。 反例：`"中长焦人像;长焦"`（不要用分号字符串，要数组）。

5. **volume_code**（必填）
   - 是什么：该术语属于哪个卷，`V01`–`V15`（新增卷先 add-volume）。
   - 正例：`"V01"`。 反例：`"1"`、`"V1"`、未注册的 `"V99"`。

6. **category**（必填 = 分类路径）
   - 是什么：术语在分类树中的完整路径，用 ` / `（空格-斜杠-空格）分隔，深度不限。
   - 怎么写：**第一段必须是该卷在 `config/volumes.json` 里声明的顶层分类**（见第七节）；中间可任意分支；**末端是原子叶子的上一层分类**（叶子=术语本身）。
   - 正例：`"镜头与光学 / 焦距 / 定焦镜头"`（该卷术语「85mm人像」挂在这条路径下）。
   - 反例：`"焦距,定焦"`（用了逗号）、`"定焦镜头"`（单层、首段不在 config 声明）、`"镜头与光学 /  / 定焦"`（空层）。

7. **definition_long**（必填）
   - 是什么：术语的**详细解释**——讲清它是什么、原理/由来、对画面的影响、典型用法。
   - 怎么写：建议 **100–200 字**（硬下限 1 字，但请写充分）；**禁止复读术语名**，**禁止占位**（待补充/TODO/简短定义）。
   - 正例：`"85mm 定焦镜头透视压缩适中、背景虚化柔美，是人像摄影最常用焦段之一，能让主体从背景中干净分离。"`
   - 反例：`"85mm人像"`（复读名字）、`"85mm人像。"`、`"待补充"`、`"这是AI高频术语。"`（套话无信息）。

8. **visual_effect**（必填）
   - 是什么：这个术语**落在画面上的视觉表现**（看起来是什么样）。
   - 正例：`"背景压缩、奶油散景、主体突出立体。"` 反例：留空、写成用途而非视觉。

9. **prompt_usage**（必填）
   - 是什么：**在提示词里怎么用**——用于什么场景、强调什么、配合哪些词/参数。
   - 正例：`"用于人像，强调压缩感与散景，配合大光圈。"` 反例：留空、与 definition 重复。

10. **use_cases**（必填，数组）
    - 是什么：适用场景列表。 怎么写：数组，至少 1 项。
    - 正例：`["人像","时尚"]`。 反例：`"人像;时尚"`（要数组）、`[]`（不能空）。

11. **related_terms**（选填，数组）：相关术语的中文名。正例：`["50mm标准","散景"]`。
12. **confused_with**（选填，数组）：易混淆术语。正例：`["135mm长焦"]`。
13. **tags**（必填，数组）：标签，至少 1 项。正例：`["焦距","定焦","85mm"]`。反例：`[]`。
14. **source_refs**（选填）：来源说明，默认 `"整理"`。
15. **status**（必填）：取值仅限 `draft` / `review` / `published` / `deprecated`；一般写 `"published"`。
16. **version**（必填）：版本号，一般写 `"V1.0"`。

## 四、分类路径规则（最容易出错，重点看）

- **结构**：`卷(体系) → 顶层分类 → 子分支 … → 原子叶子术语`。每个节点**要么继续分支、要么挂术语**，不要在同一层既放概述词又放术语。
- **顶层段必须 ∈ config 声明**：否则接口直接拒绝并列出该卷允许的顶层分类。要新顶层就先 add-volume 或在 config 增补，不要私自造。
- **原子化**：笼统词是「分支名」不是「术语」。例：`镜头与光学 / 焦距 / 定焦镜头 / 50mm标准` —— 前面都是分支，只有 `50mm标准` 是术语（zh_term）。
- **互斥靠结构**：把互斥取值放进同一最深分类下（如 35mm/50mm/85mm 同在 `…/定焦镜头`），它们天然互为可选项；前端篮里并存会标红（仅提示，不阻断）。
- **查重**：接口在新增时会比对——同卷 zh_term 完全相同→拒绝；高度相似→警告（请确认不是重复）。已有的近义重复不强制合并，但新增时请尽量复用已有分类与命名。

## 五、三种场景的完整示例

### A. 给已有分支加原子术语 / 给已有卷加新分支（同一命令）
`terms.json`：
```json
[
  {
    "zh_term": "星芒效果", "en_term": "Sunstar",
    "volume_code": "V01", "category": "镜头与光学 / 光学缺陷与效果",
    "definition_long": "小光圈下点光源经光圈叶片衍射形成的放射状星芒，叶片数决定芒线数量，常用于夜景灯光与逆光太阳。",
    "visual_effect": "点光源化为放射状星芒、画面通透锐利。",
    "prompt_usage": "夜景或逆光需要星芒时加入，配合小光圈 f/11–f/16。",
    "use_cases": ["夜景", "逆光", "建筑"],
    "related_terms": ["小光圈"], "confused_with": ["眩光"],
    "tags": ["镜头", "光学", "星芒"], "status": "published", "version": "V1.0"
  }
]
```
```bash
python scripts/ingest.py check terms.json      # 先看校验
python scripts/ingest.py add-terms terms.json  # 通过后入库
```

### B. 新增整卷
`volume.json`（见 `docs/templates/volume.template.json`）：
```json
{ "code": "V16", "sequence": 16, "title": "你的新卷标题",
  "target_terms": 600, "purpose": "一句话说明这个卷收录什么。",
  "categories": ["顶层分类1", "顶层分类2", "顶层分类3"] }
```
```bash
python scripts/ingest.py add-volume volume.json   # 注册卷
# 然后照 A 给该卷 add-terms，category 首段取自上面的 categories
```

## 六、校验会拦下哪些错误（出现即整批不写入）

- 缺必填字段 / 必填为空（zh_term、en_term、volume_code、category、definition_long、visual_effect、prompt_usage、use_cases、tags、status、version）。
- term_uid 格式错、前缀与卷不符、重复。
- volume_code 未在 config 注册。
- category 顶层段不在该卷 config 声明里 / 含空层 / 用了逗号。
- definition_long 复读术语名、或含占位词（待补充/TODO/TBD/简短定义）。
- 同卷 zh_term 重复。
- status 不在四种取值内。

警告（不阻断，但请核对）：同卷出现高度相似的 zh_term（疑似重复）。
