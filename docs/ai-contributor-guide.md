# AI 供稿指南 — AI视觉设计与提示词工程百科

> 本文件供 AI 模型（Claude / GPT / Gemini 等）阅读，用于**按规范批量补全卷册/体系的术语内容**。
> 读完即可上手。**唯一需要编辑的文件是 `data/raw/terms_seed.csv`。**

---

## 一、项目是什么

面向 15 卷、目标约 10,000 条专业「视觉/图像」术语的结构化知识库。每条术语同时服务两件事：

1. **百科**：中英文名 + 定义 + 视觉表现。
2. **AI 生图提示词**：可直接粘进 Stable Diffusion / Midjourney / DALL·E 等的中英双语提示词。

数据流：`data/raw/terms_seed.csv` → `build_kb.py` → SQLite 主库（唯一权威源）→ API / 前端 / 导出物。
**你只管往 CSV 追加规范的新行**，其余自动生成。

当前已有约 1545 条（15 卷均有数据），属"骨架已铺、需继续加密"阶段。

## 二、边界：必须 / 严禁（先读这条，违反会被校验拒绝或破坏一致性）

### ✅ 必须

| # | 规则 |
|---|---|
| 1 | **只编辑 `data/raw/terms_seed.csv`，只「追加」新行**。不得修改或删除任何已有行。 |
| 2 | 每行 **21 列、顺序固定、单行写完**（字段内不能换行）。编码 UTF-8 BOM。 |
| 3 | `term_uid` = `V{两位卷号}_T{四位序号}`，**卷内唯一、递增**。追加前先查该卷现有最大号，从 +1 开始。 |
| 4 | `category` 是**路径**，用 ` / `（空格-斜杠-空格）分隔，深度不限。第一段须是该卷的顶层分类，末端是**原子叶子**。例：`镜头与景深 / 焦距 / 定焦镜头 / 85mm人像`。 |
| 5 | **术语原子化**：叶子必须是「能直接出图的最具体取值」。颜色要到深浅浓淡，焦距要到具体毫米。 |
| 6 | **中英双语提示词都填且=术语本身**：`positive_prompt`(英) + `positive_prompt_cn`(中) 一一对应术语。例：`冷蓝色调` → `cool blue tone` / `冷蓝色调`。 |
| 7 | 必填字段非空：term_uid, zh_term, en_term, volume_code, category, definition_short, definition_long, visual_effect, prompt_usage, positive_prompt, positive_prompt_cn, use_cases, tags, status, version。 |
| 8 | 多值字段（aliases / use_cases / related_terms / confused_with / tags）用**英文分号 `;`** 分隔。 |
| 9 | 含英文逗号的字段，整段**用双引号包裹**（用 Python `csv` 库写会自动处理，强烈建议）。 |
| 10 | `status` 用 `published`，`version` 用 `V1.0`。 |

### ⛔ 严禁

| # | 禁止 |
|---|---|
| 1 | **不填负向提示词**：`negative_prompt` / `negative_prompt_cn` 两列保留但**留空**（本项目不用负向）。 |
| 2 | **不要笼统词当叶子**：`焦距` / `颜色` / `色温` / `镜头畸变` 这类是「分类/分支」，不是术语。要继续分支到 `85mm人像` / `深红色` / `暖色温3200K` / `桶形畸变`。 |
| 3 | **不要把概述词和子分类并列同层**。反例：`焦距与视角` 下同时放 `焦距`(术语) 和 `定焦镜头`(子类)。正解：`焦距 / 定焦镜头 / 50mm标准`。 |
| 4 | 不得 `category` 用逗号、或只写单层。 |
| 5 | 不得新增列 / 字段 / 互斥组等；列结构固定 21 列。 |
| 6 | 不得重复 `term_uid`；同一卷内不得重复 `zh_term`。 |
| 7 | `positive_prompt` 不要写解释性句子，要「逗号分隔的关键词组合」。 |
| 8 | 不要编造不存在的术语；定义须专业准确。 |

> **冲突如何规避**：不靠字段，**靠分支结构**——把互斥的取值放进同一最深分类下（如 35mm/50mm/85mm 同在 `…/定焦镜头`），它们天然互斥；前端篮中若同时选中会自动标红提示。

## 四、用法：分步流程

**第 0 步｜读规则**：读完本文件的「边界」与「契约」。

**第 1 步｜选一个卷**：一次专注一卷（如 V06 灯光与色彩）。卷册见第五节。

**第 2 步｜查现状，避免重复**：
- 看已有术语与最大 UID：`GET /api/terms?volume=V06&page_size=200`（或直接读 CSV）
- 查某术语是否已存在：`GET /api/search?q=暖色温`
- 看哪卷哪分类还缺：`GET /api/meta`

**第 3 步｜先规划「拓扑」再填**：为该卷画出分类路径树（体系顶层分类 → 子类 → … → 原子叶子），把**可枚举维度逐一列全**。例：色温列全 1800K–10000K 各档；焦距列全 14/24/35/50/85/135/200mm；颜色按色相 × 深浅浓淡列全。

**第 4 步｜逐叶子写行**：每个原子叶子写一行 21 列，UID 从该卷现有最大号 +1。

**第 5 步｜写入 CSV（推荐用脚本，自动转义）**：
强烈建议仿照仓库里 `scripts/gen_v01.py … gen_v15.py` 的"生成器 + 合并模式"写法（读 CSV → 剔除本卷旧行 → 追加新行 → 重编号 → 写回，用 `csv.DictWriter` 自动转义、且可反复重跑不污染他卷）。也可手工追加到文件末尾，但务必保证逗号转义与列数正确。

**第 6 步｜构建 + 校验**：
```bash
./manage.sh build      # = python scripts/build_kb.py，重建库
./manage.sh validate   # = python scripts/validate_kb.py，校验
```
校验须 `Validation passed.`，否则按提示修正（常见：UID 重复、同卷 zh_term 重复、列数不对）。

**第 7 步｜核对**：`GET /api/meta` 看计数变化，`GET /api/search?q=...` 抽查新术语。

## 七、只读 API（核对现状用）

| 接口 | 用途 |
|------|------|
| `GET /api/meta` | 15 卷结构 + 各卷分类与计数（看缺口） |
| `GET /api/terms?volume=V06&page_size=200` | 某卷已有术语（查最大 UID、避免重复） |
| `GET /api/terms/{term_uid}` | 单条详情 |
| `GET /api/search?q=关键词` | 全文搜索（中文走 trigram，≥3字；短词走 LIKE） |
| `GET /api/stats` | 各卷完成度 |

## 八、常见错误对照

| 错误 | 纠正 |
|------|------|
| 多值用逗号 | 用英文分号 `;`；含逗号字段整段加双引号 |
| term_uid 重复 / 同卷 zh_term 重复 | 追加前查最大号；卷内中文名唯一 |
| category 单层或用逗号 | category 是 ` / ` 路径，分支到原子叶子 |
| 术语太笼统（焦距/颜色/色温） | 原子化到具体值（85mm/深红色/暖色温3200K） |
| 概述词与子分类并列同层 | 概述词应是分支名，叶子才是术语 |
| 只填英文提示词 / 填了负向 | 中英双语都填且对应；负向两列留空 |
| positive_prompt 写成句子 | 改为逗号分隔关键词组合 |
| 缺必填 / status 拼错 / 行内换行 | 必填非空；status∈draft/review/published；单行 |
