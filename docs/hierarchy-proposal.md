# 四层分类体系方案（体系 › 分类 › 子类 › 术语）

> 设计目标：覆盖所有图像/视觉相关领域，兼顾摄影、影视、绘画、设计、3D、AI 等不同职业与专业；
> 每个「子类」是一组强相关术语的归宿（建议每子类容纳 15–40 条术语），为 1 万条目标留足扩展空间。
> 命名规范：分类 = 一级专业方向；子类 = 可独立成章的知识单元。下面每个分类后列出建议子类。

## V01 摄影体系（目标 500）
- 摄影基础：摄影术语 / 拍摄模式 / 测光与对焦 / 持机与稳定
- 相机与感光：相机类型 / 传感器与画幅 / 感光特性(ISO噪点) / 文件格式
- 曝光控制：光圈 / 快门速度 / 感光度 / 曝光补偿与测光 / 动态范围
- 镜头与景深：焦距与视角 / 景深控制 / 镜头类型 / 光学缺陷(畸变/色散)
- 构图与机位：构图法则 / 拍摄角度 / 机位距离 / 画面平衡
- 摄影类型：人像 / 风光 / 街拍纪实 / 产品静物 / 微距与特殊

## V02 电影摄影体系（目标 800）

- 灯光与影调：布光法 / 影调风格 / 光质与方向 / 实用光源
- 镜头语言：镜头调度 / 焦点运用 / 长镜头与蒙太奇 / 主客观视角
- 电影格式：画幅与变形宽银幕 / 帧率 / 胶片与数字 / LOG与RAW
- 片场工作流：部门分工 / 灯光器材 / 现场监看 / 拍摄计划

- 笔触与肌理：笔法 / 肌理技法 / 媒介质感 / 留白与堆叠

## 数据落地方式（确认后实施）
1. CSV 新增可选列 `subcategory`（子类），位于 `category` 与术语之间；旧数据 `subcategory` 留空时回退为 3 层。
2. `config/volumes.json` 增加 `category → subcategories` 映射，作为子类白名单与排序依据。
3. `build_kb.py` 用 `categories.parent_id` 建立「分类→子类」父子关系（schema 已支持）。
4. API `/api/meta` 返回嵌套子类；前端树与面包屑渲染 4 层。
5. 现有 243 条术语回填 `subcategory`，使深度立即可见。
